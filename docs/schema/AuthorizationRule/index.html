AuthorizationRule

Schema: http://rustybeam.net/AuthorizationRule

An AuthorizationRule defines access control policies for HTTP requests based on path patterns, HTTP methods, user roles, usernames, and CSS selectors.

Property Type Cardinality Description
path https://schema.org/Text 1 Path pattern to match (supports wildcards and {username} placeholder, e.g., "/admin/*", "/users/{username}/*").
method https://schema.org/Text 1 HTTP method(s) to match (supports wildcards and comma-separated values, e.g., "GET", "PUT,POST,DELETE", "*").
role https://schema.org/Text 0..1 Required user role for access (e.g., "administrators", "users").
username https://schema.org/Text 0..1 Required username for access (supports {username} placeholder for matching authenticated user).
selector https://schema.org/Text 0..1 CSS selector to restrict access to specific HTML elements (e.g., "ul.entry").
action https://schema.org/Text 1 Action to take when rule matches ("allow" or "deny").

Examples

Administrator Access

<tr itemprop="rule" itemscope itemtype="http://rustybeam.net/AuthorizationRule">
    <td itemprop="path">/admin/*</td>
    <td itemprop="method">*</td>
    <td itemprop="role">administrators</td>
    <td itemprop="action">allow</td>
</tr>
