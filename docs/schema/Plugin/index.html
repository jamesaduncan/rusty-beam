Plugin Schema

The Plugin schema defines the configuration for individual plugins in the Rusty Beam processing pipeline. Each plugin is loaded as a dynamic library and can have plugin-specific configuration properties.

Schema Definition

Schema URL: http://rustybeam.net/Plugin

Properties

Property Type Cardinality Description
library URL 1 The path to the plugin shared library file using file:// protocol (e.g., "file://./plugins/basic-auth.so")
plugin Plugin 0..n Nested plugins that form a sub-pipeline within this plugin (used for scoped plugin execution)
realm Text 0..1 Authentication realm for basic-auth plugin (e.g., "Admin Area")
authfile URL 0..1 Path to authentication or authorization file for basic-auth and authorization plugins
directory Text 0..1 Path prefix for directory plugin to scope nested plugins (e.g., "/admin")
logfile URL 0..1 Path to log file for access-log plugin
rulesfile URL 0..1 Path to redirect rules file for redirect plugin
