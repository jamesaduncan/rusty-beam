RedirectRules

Schema: http://rustybeam.net/RedirectRules

RedirectRules is a container schema that holds a collection of RedirectRule items. It represents a table or list of URL redirection configurations that the redirect plugin processes in order.

Property Type Cardinality Description
rule http://rustybeam.net/RedirectRule 0..* A redirect rule entry that defines a URL redirection pattern. Multiple rules can be specified and are processed in order.

Example Usage

<table itemscope itemtype="http://rustybeam.net/RedirectRules">
    <tbody>
        <!-- Simple redirect -->
        <tr itemprop="rule" itemscope itemtype="http://rustybeam.net/RedirectRule">
            <td itemprop="from">/old-page.html</td>
            <td itemprop="to">/new-page.html</td>
            <td itemprop="status">301</td>
        </tr>
        
        <!-- Pattern-based redirect -->
        <tr itemprop="rule" itemscope itemtype="http://rustybeam.net/RedirectRule">
            <td itemprop="from">/blog/(.+)</td>
            <td itemprop="to">/articles/$1</td>
            <td itemprop="status">301</td>
        </tr>
        
        <!-- External redirect -->
        <tr itemprop="rule" itemscope itemtype="http://rustybeam.net/RedirectRule">
            <td itemprop="from">/external</td>
            <td itemprop="to">https://example.com</td>
            <td itemprop="status">302</td>
        </tr>
    </tbody>
</table>

Processing Order

Rules are processed in the order they appear in the table. The first matching rule will be applied, and subsequent rules will not be evaluated. This allows for specific rules to take precedence over more general patterns.

Pattern Matching

The redirect plugin supports regular expression patterns in the "from" field:

  • Exact matches: /old-page.html
  • Capture groups: /blog/(.+) captures content after /blog/
  • Replacements: Use $1, $2, etc. in the "to" field to reference capture groups

Integration with Redirect Plugin

The RedirectRules table is typically included in a host configuration file and loaded by the redirect plugin. The plugin reads these rules and applies them to incoming HTTP requests, generating appropriate redirect responses.
