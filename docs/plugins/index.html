Rusty Beam Plugins

Rusty Beam's plugin architecture allows extending the server with custom functionality. Plugins are loaded dynamically and process requests in a pipeline.

Core Plugins

Selector-Handler

Enables CSS selector-based HTML manipulation via Range headers. The foundation of Rusty Beam's unique API.

Range: selector=.content

File-Handler

Serves static files from the file system with proper MIME types and security checks.

GET /index.html

WebSocket

Real-time bidirectional communication with automatic document change subscriptions.

ws://host/document.html

Authentication & Authorization

Basic-Auth

HTTP Basic Authentication with support for multiple password encryption methods.

Authorization: Basic ...

Authorization

Role-based access control with flexible rules for paths and methods.

Require: role admin

Request Processing

CORS

Cross-Origin Resource Sharing support with configurable origins and methods.

Access-Control-Allow-Origin: *

Compression

Automatic response compression with gzip and deflate support.

Content-Encoding: gzip

Security-Headers

Adds security headers like CSP, HSTS, and X-Frame-Options.

Strict-Transport-Security

Traffic Management

Rate-Limit

Token bucket rate limiting per IP address or globally.

X-RateLimit-Remaining: 99

Redirect

URL redirection with pattern matching and permanent/temporary redirects.

301 Moved Permanently

Monitoring & Logging

Access-Log

Apache/Nginx compatible access logging with customizable formats.

127.0.0.1 - - [01/Jan/2024...]

Health-Check

Health check endpoints for monitoring and load balancers.

GET /health → 200 OK

Error-Handler

Custom error pages and centralized error handling.

404.html, 500.html

Additional Plugins

Directory

Path-based plugin routing with nested plugin execution. Apply different plugin pipelines to different URL paths.

directory: /admin → auth + files
